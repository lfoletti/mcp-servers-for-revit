M-Long-Bulk — same thirty wall height edits as M-Long, but issued in bulk.

You will perform thirty wall height edits in two batches with a mid-run
differential between them. Each batch must be issued as a SINGLE bulk
operation if your project-memory tools expose one (look for something
like batch_set_parameters or an equivalent that accepts a list of
{element_id, param, value} in one call). If — and only if — no such
tool exists, fall back to one edit at a time, but say so explicitly.

  ──── Batch A (15 edits) ────
  Edits 1-10: walls 1 through 10, set height to 2.80, 2.81, …, 2.89 m
                (i.e. wall N → height 2.80 + (N-1)/100 for N in 1..10).
  Edits 11-15: walls 11 through 15, set height to 3.10, 3.12, 3.14, 3.16,
                3.18 m respectively.
  Issue Batch A as ONE bulk call with all 15 operations.

  ──── Mid-run checkpoint, AFTER Batch A ────
  Query your project-memory tools for the differential since the start of
  this scenario. Report concisely: how many walls were modified, the list
  of (wall id, new height), and the turn-or-step count if your tools
  expose one. Do NOT re-query the full project — just the diff.

  ──── Batch B (15 edits) ────
  Edits 16-25: walls 16 through 25, set height to 2.60, 2.61, …, 2.69 m.
  Edits 26-30: walls 26 through 30, set height to 3.20, 3.21, 3.22, 3.23,
                3.24 m.
  Issue Batch B as ONE bulk call with all 15 operations.

End with a final differential summary covering all 30 edits, plus a
one-line tally: "<applied>/30 edits persisted in <K> bulk call(s)".
